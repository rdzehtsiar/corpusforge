# Grammar Workflow Demo

CorpusForge Milestone 8 includes fixture-based grammar generation for Markdown and JSON in `valid`, `near-valid`, and `malformed` modes. These workflows are deterministic for the same CorpusForge version, seed, grammar, mode, Unicode composition mode, and case count, but they are not Markdown or JSON conformance suites.

The demo harnesses in `examples/` read UTF-8 text from stdin and exit with a nonzero status for intentionally narrow parser- or renderer-style checks:

- `reject_open_markdown_fence.rs` rejects unclosed fenced code blocks and open inline links or images.
- `reject_malformed_json.rs` rejects simple JSON structural problems such as unbalanced delimiters and trailing commas.

## Build the Demo Harnesses

From the repository root on Windows PowerShell:

```powershell
New-Item -ItemType Directory -Force target
rustc examples\reject_open_markdown_fence.rs -o target\reject-open-markdown-fence-demo.exe
rustc examples\reject_malformed_json.rs -o target\reject-malformed-json-demo.exe
```

On Unix-like shells:

```sh
mkdir -p target
rustc examples/reject_open_markdown_fence.rs -o target/reject-open-markdown-fence-demo
rustc examples/reject_malformed_json.rs -o target/reject-malformed-json-demo
```

## Generate Grammar Samples

Generate malformed Markdown cases:

```powershell
cargo run -p corpusforge-cli -- gen --grammar markdown --grammar-mode malformed --cases 4 --seed 1337 --out target\grammar-markdown-demo.md
```

Generate malformed JSON cases:

```powershell
cargo run -p corpusforge-cli -- gen --grammar json --grammar-mode malformed --cases 4 --seed 1337 --out target\grammar-json-demo.txt
```

The same commands on Unix-like shells use slash-separated paths:

```sh
cargo run -p corpusforge-cli -- gen --grammar markdown --grammar-mode malformed --cases 4 --seed 1337 --out target/grammar-markdown-demo.md
cargo run -p corpusforge-cli -- gen --grammar json --grammar-mode malformed --cases 4 --seed 1337 --out target/grammar-json-demo.txt
```

Valid and near-valid modes can be generated by changing `--grammar-mode`:

```powershell
cargo run -p corpusforge-cli -- gen --grammar markdown --grammar-mode valid --cases 4 --seed 1337 --out target\grammar-markdown-valid.md
cargo run -p corpusforge-cli -- gen --grammar json --grammar-mode near-valid --cases 4 --seed 1337 --out target\grammar-json-near-valid.txt
```

## Run the Harnesses

On Windows PowerShell, pipe the generated files into the harnesses:

```powershell
Get-Content -Raw target\grammar-markdown-demo.md | target\reject-open-markdown-fence-demo.exe
Get-Content -Raw target\grammar-json-demo.txt | target\reject-malformed-json-demo.exe
```

The malformed demo commands are expected to return a nonzero exit status when a generated case trips the narrow check. To inspect the status in PowerShell:

```powershell
Get-Content -Raw target\grammar-json-demo.txt | target\reject-malformed-json-demo.exe
$LASTEXITCODE
```

On Unix-like shells:

```sh
./target/reject-open-markdown-fence-demo < target/grammar-markdown-demo.md
./target/reject-malformed-json-demo < target/grammar-json-demo.txt
echo $?
```

## Current Limitations

These harnesses are deliberately small offline demos. The Markdown harness does not parse Markdown; it checks only for an unclosed fenced code block and a small inline link/image shape. The JSON harness is a structural rejector for line-oriented generated cases; it is not a full JSON parser and may accept invalid JSON that does not violate its simple checks.

Milestone 8 grammar generation currently demonstrates built-in fixture-style Markdown and JSON cases for local parser or renderer harness workflows. Treat broader grammar coverage, conformance claims, structure-aware shrinking, and dedicated grammar CI reports as unsupported unless future tests and documentation name the verified behavior.
